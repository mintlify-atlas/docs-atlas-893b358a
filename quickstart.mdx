## Installation

<Steps>
  <Step title="Clone the repository">
    Clone the Super Hexagon project to your local machine:
    ```bash
    git clone <repository-url>
    cd superhexagon
    ```
  </Step>

  <Step title="Install dependencies">
    Install the required packages using npm or yarn:
    
    <CodeGroup>
    ```bash npm
    npm install
    ```
    
    ```bash yarn
    yarn install
    ```
    </CodeGroup>
    
    The project uses the following key dependencies:
    - **Next.js 13.4.12** - React framework
    - **React 18.2.0** - UI library
    - **Redux Toolkit** - State management
    - **Framer Motion** - Animations
    - **Tailwind CSS** - Styling
  </Step>

  <Step title="Run the development server">
    Start the development server:
    
    <CodeGroup>
    ```bash npm
    npm run dev
    ```
    
    ```bash yarn
    yarn dev
    ```
    </CodeGroup>
    
    Open [http://localhost:3000](http://localhost:3000) in your browser to play the game.
  </Step>
</Steps>

<Note>
The development server features hot module replacement, so changes to your code will update automatically without refreshing the page.
</Note>

## Available Scripts

The project includes the following npm scripts defined in `package.json`:

| Script | Command | Description |
|--------|---------|-------------|
| `dev` | `next dev` | Starts the development server on port 3000 |
| `build` | `next build` | Creates an optimized production build |
| `start` | `next start` | Runs the production server |
| `lint` | `next lint` | Runs ESLint to check code quality |

## Building for Production

To create an optimized production build:

<Steps>
  <Step title="Build the application">
    ```bash
    npm run build
    ```
    
    This command compiles the application and optimizes it for best performance.
  </Step>

  <Step title="Start the production server">
    ```bash
    npm run start
    ```
    
    The production server will start on [http://localhost:3000](http://localhost:3000).
  </Step>
</Steps>

## Project Structure

The Super Hexagon codebase follows a component-based architecture:

```
src/
├── app/
│   ├── layout.tsx          # Root layout component
│   └── page.tsx            # Main page (renders Game component)
├── components/
│   ├── atoms/              # Smallest reusable components
│   │   ├── Audio/          # Audio controller and playback
│   │   └── Score/          # Score display component
│   ├── molecules/          # Composite components
│   │   ├── Line/           # Obstacle line with collision detection
│   │   └── Player/         # Player triangle with controls
│   └── organisms/          # Complex feature components
│       ├── Game/           # Main game orchestrator
│       ├── Menu/           # Start menu and game over screen
│       └── Superhexagon/   # Game board with hexagon and obstacles
├── redux/
│   ├── slices/             # Redux state slices
│   │   └── game.ts         # Game state (score, player position, etc.)
│   ├── selectors/          # Redux selectors for state access
│   └── store.ts            # Redux store configuration
├── helpers/
│   ├── playerControllers.ts    # Keyboard and touch input handlers
│   ├── collisionControllers.ts # Collision detection algorithms
│   └── rayCasting.ts          # Ray casting utilities
└── styles/
    └── globals.css         # Global styles and Tailwind imports
```

### Component Architecture

The application follows atomic design principles:

- **Atoms**: Basic building blocks (`Score`, `Audio`)
- **Molecules**: Composite components (`Player`, `Line`)
- **Organisms**: Complex features (`Game`, `Menu`, `SuperHexagon`)

### State Management

The game uses **Redux Toolkit** for centralized state management:

```typescript src/redux/slices/game.ts
const initialState: PlayerState = {
  isPLaying: false,
  player: { hitbox1: null, hitbox2: null, hitbox3: null },
  angle: '0',
  score: 0,
  volume: 0.4,
}
```

Key state includes:
- `isPlaying` - Game state (playing/paused)
- `player` - Player hitbox positions for collision detection
- `angle` - Board rotation angle
- `score` - Current and best scores
- `volume` - Audio volume setting
