## Overview

The `SuperHexagon` component is the core game engine that renders the playable hexagon arena. It manages the rotating visual field, spawns obstacles at intervals, tracks player rotation angle, and handles collision detection.

## Location

`src/components/organisms/Superhexagon/SuperHexagon.tsx`

## Key Features

- **Rotating Arena**: 360-degree continuous rotation using Framer Motion
- **Dynamic Obstacle Spawning**: Interval-based line generation with random gaps
- **Angle Tracking**: Real-time rotation angle updates to Redux
- **Six Triangle Zones**: Hexagonal structure divided into 6 triangular sections
- **Collision Audio**: Pre-loaded game over sound effect

## Obstacle Spawning Logic

The component spawns obstacles every 1.8 seconds using a sophisticated randomization system:

```tsx
const handleSpawn = useCallback(() => {
  const randomId = Math.floor(Math.random() * 6)

  triangles.map((triangle, idx) => {
    if (idx === randomId) return // Skip one random triangle = gap
    const div = document.createElement('div')
    div.className = 'h-0'
    createRoot(div).render(
      <StoreProvider>
        <Line audio={audio} />
      </StoreProvider>
    )
    triangle.current?.appendChild(div)
  })
}, [triangles, audio])

useEffect(() => {
  const spawnIntervalId = setInterval(handleSpawn, 1800)
  dispatch(setSpawnIntervalId(spawnIntervalId))
}, [dispatch, handleSpawn])
```

### Spawning Process

1. **Random Gap Selection**: Choose a random number 0-5 to determine which triangle section will be the gap
2. **Line Creation**: For each non-gap triangle, create a new div container with `h-0` class
3. **React Root Injection**: Use `createRoot()` to render a `<Line>` component with Redux provider
4. **DOM Append**: Attach the Line to the triangle ref's DOM node
5. **Interval Storage**: Store interval ID in Redux for cleanup on game over

<Note>
  Each Line component spawns with its own StoreProvider to ensure Redux connectivity. The Line animates inward and self-destructs upon reaching the center hexagon.
</Note>

## Angle Tracking System

The component continuously monitors the rotation angle and syncs it to Redux:

```tsx
useEffect(() => {
  const playerIntervalId = setInterval(() => {
    const transform = ref.current?.style?.transform
    if (transform)
      dispatch(
        updateAngle({
          angle: transform
            ?.match(/rotate\(.+deg\)/gm)?.[0]
            .split('.')[0]
            ?.match(/([0-9]+)/gm)?.[0],
        })
      )
  }, 20) // Updates every 20ms = 50 times per second
  dispatch(setPlayerIntervalId(playerIntervalId))
}, [dispatch])
```

### Angle Extraction Process

1. **Read Transform**: Access the CSS `transform` style from the rotating container ref
2. **Regex Match**: Extract the `rotate(Xdeg)` value using regex pattern `/rotate\(.+deg\)/gm`
3. **Parse Number**: Split on decimal and extract integer portion of degrees
4. **Dispatch to Redux**: Update global `angle` state for collision detection
5. **High Frequency**: Updates 50 times per second (every 20ms) for smooth tracking

<Warning>
  The angle is stored as a string in Redux due to the parsing logic. Components using this value should convert it to a number when needed.
</Warning>

## Rotation Animation

The arena uses Framer Motion for smooth, infinite rotation:

```tsx
<motion.div
  ref={ref}
  className='box relative'
  initial={{ rotate: 0 }}
  animate={{ rotate: 360 }}
  transition={{ repeat: Infinity, duration: 30, ease: 'linear' }}
>
  {/* hexagon structure */}
</motion.div>
```

<ParamField path="initial" type="object">
  Starting rotation angle: `{ rotate: 0 }`
</ParamField>

<ParamField path="animate" type="object">
  Target rotation: `{ rotate: 360 }` (full circle)
</ParamField>

<ParamField path="transition" type="object">
  - `repeat: Infinity` - Never stops rotating
  - `duration: 30` - One full rotation takes 30 seconds
  - `ease: 'linear'` - Constant speed (no acceleration)
</ParamField>

## Triangle Container System

The six triangular sections are rendered dynamically:

```tsx
{triangles.map((triangle, idx) => (
  <div
    key={idx}
    ref={triangle}
    className='triangle top-[calc(50vh_-_260px)]'
  ></div>
))}
```

Each triangle:
- Uses the `.triangle` CSS class for shape styling
- Positioned `260px` above viewport center
- Serves as a container for spawned Line components
- Maintains its own ref for targeted obstacle injection

## Player Component

The SuperHexagon renders the Player component outside the rotating container:

```tsx
<Player />
```

This separation ensures the player triangle remains stationary while the hexagon rotates beneath it. The Player component handles:
- User input (left/right arrow keys)
- Position updates based on rotation state
- Collision detection with Line obstacles

## Cleanup and Memory Management

The component stores interval IDs in Redux to enable proper cleanup:

1. **Spawn Interval**: Cleared when game ends to stop obstacle generation
2. **Player Interval**: Cleared when game ends to stop angle updates
3. **Line Components**: Self-destruct after animation completes or collision occurs

<Note>
  The interval cleanup happens in the collision handler (Line component) when `pause()` action is dispatched. This prevents memory leaks and stops game logic immediately on game over.
</Note>

## Performance Considerations

### Memoization

- **triangles array**: Memoized with `useMemo` to prevent re-creation
- **audio element**: Memoized to avoid re-loading sound file
- **handleSpawn callback**: Memoized with `useCallback` to stabilize dependency array

### Interval Frequencies

- **Spawn interval**: 1800ms (every 1.8 seconds)
- **Angle update interval**: 20ms (50 updates per second)
- **Rotation duration**: 30 seconds per full cycle

### DOM Manipulation

The component uses direct DOM manipulation (`createElement`, `appendChild`) for obstacle spawning rather than React state. This decision:
- Avoids re-rendering the entire hexagon on each spawn
- Allows precise control over Line component lifecycle
- Reduces React reconciliation overhead
