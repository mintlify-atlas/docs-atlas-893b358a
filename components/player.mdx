## Overview

The Player component renders the controllable player cursor in Super Hexagon. It manages keyboard and touch input, rotation mechanics, and provides three hitbox points for collision detection. The component dispatches position updates to Redux for use by the collision system.

## Component Structure

The Player component is located at `src/components/molecules/Player/Player.tsx` and implements:

- Keyboard controls (Arrow Left/Right)
- Touch controls for mobile devices (screen width < 800px)
- Rotation mechanics at 2 degrees per frame
- Three hitbox reference points for precise collision detection
- Redux integration for position state management

## Control System

### Keyboard Controls

The component registers global keyboard event listeners:

```tsx
useEffect(() => {
  window.addEventListener('keydown', (e) =>
    keyDownHandler({ key: e.key, setLeftHeld, setRightHeld })
  )
  window.addEventListener('keyup', (e) =>
    keyUpHandler({ key: e.key, setLeftHeld, setRightHeld })
  )
  return () => {
    window.removeEventListener('keydown', (e) =>
      keyDownHandler({ key: e.key, setLeftHeld, setRightHeld })
    )
    window.removeEventListener('keyup', (e) =>
      keyUpHandler({ key: e.key, setLeftHeld, setRightHeld })
    )
  }
}, [])
```

**Key Mappings:**
- **ArrowLeft**: Rotate counterclockwise
- **ArrowRight**: Rotate clockwise

### Control Handlers

From `src/helpers/playerControllers.ts:3`:

```tsx
export const keyDownHandler = ({
  key,
  setLeftHeld,
  setRightHeld,
}: {
  key: string
  setLeftHeld: (value: SetStateAction<boolean>) => void
  setRightHeld: (value: SetStateAction<boolean>) => void
}) => {
  if (key === 'ArrowLeft') {
    setLeftHeld(true)
    setRightHeld(false)
  }
  if (key === 'ArrowRight') {
    setRightHeld(true)
    setLeftHeld(false)
  }
}
```

The handlers ensure only one direction can be active at a time, preventing conflicting inputs.

### Rotation Logic

The player rotates at 2 degrees per frame based on held keys:

```tsx
export const playerController = ({
  leftHeld,
  rightHeld,
  degres,
  setDegres,
  el,
}: {
  leftHeld: boolean
  rightHeld: boolean
  degres: number
  setDegres: (value: SetStateAction<number>) => void
  el: HTMLDivElement | null
}) => {
  if (leftHeld && el) {
    setDegres(degres - 2)
    el.style.transform = `rotate(${degres}deg)`
  }
  if (rightHeld && el) {
    setDegres(degres + 2)
    el.style.transform = `rotate(${degres}deg)`
  }
}
```

The rotation updates every 10ms via a setTimeout:

```tsx
useEffect(() => {
  setTimeout(() => {
    playerController({
      leftHeld,
      rightHeld,
      degres,
      setDegres,
      el: ref.current,
    })
  }, 10)
}, [degres, leftHeld, rightHeld])
```

## Hitbox System

### Three-Point Hitbox

The Player component maintains three hitbox points for collision detection:

```tsx
const refHitbox1 = useRef<HTMLDivElement>(null)
const refHitbox2 = useRef<HTMLDivElement>(null)
const refHitbox3 = useRef<HTMLDivElement>(null)
```

### Hitbox Positioning

The hitboxes are positioned within the player cursor:

```tsx
<div className='relative cursor bg-[red]'>
  <div
    ref={refHitbox1}
    className='absolute left-0 right-0 ml-auto mr-auto z-[99] w-[1px] min-h-[1px]'
  />
  <div
    ref={refHitbox2}
    className='absolute left-0 bottom-0 ml-auto mr-auto z-[99] w-[1px] min-h-[1px]'
  />
  <div
    ref={refHitbox3}
    className='absolute right-0 bottom-0 ml-auto mr-auto z-[99] w-[1px] min-h-[1px]'
  />
</div>
```

- **Hitbox 1**: Top center
- **Hitbox 2**: Bottom left
- **Hitbox 3**: Bottom right

### Position Updates

The component dispatches hitbox coordinates to Redux whenever the player rotates:

```tsx
useEffect(() => {
  const hitbox1 = {
    x: refHitbox1.current?.getBoundingClientRect().x ?? 0,
    y: refHitbox1.current?.getBoundingClientRect().y ?? 0,
  }
  const hitbox2 = {
    x: refHitbox2.current?.getBoundingClientRect().x ?? 0,
    y: refHitbox2.current?.getBoundingClientRect().y ?? 0,
  }
  const hitbox3 = {
    x: refHitbox3.current?.getBoundingClientRect().x ?? 0,
    y: refHitbox3.current?.getBoundingClientRect().y ?? 0,
  }

  ref.current && dispatch(updatePosition({ hitbox1, hitbox2, hitbox3 }))
}, [degres, dispatch])
```

The Line component consumes these positions for collision detection via Redux selectors.

## Touch Controls

### Mobile Detection

The component detects mobile devices on mount:

```tsx
useEffect(() => {
  setIsMobile(window.innerWidth < 800)
}, [])
```

### Touch Button Implementation

When mobile is detected, two circular buttons are rendered:

```tsx
{isMobile && (
  <>
    <button
      onTouchStart={() => {
        keyDownHandler({ key: 'ArrowLeft', setLeftHeld, setRightHeld })
      }}
      onTouchEnd={() => {
        keyUpHandler({ key: 'ArrowLeft', setLeftHeld, setRightHeld })
      }}
      className='fixed bottom-6 left-6 w-16 h-16 p-3 rounded-full border-2 border-white bg-black shadow-lg'
    ></button>
    <button
      onTouchStart={() => {
        keyDownHandler({ key: 'ArrowRight', setLeftHeld, setRightHeld })
      }}
      onTouchEnd={() => {
        keyUpHandler({ key: 'ArrowRight', setLeftHeld, setRightHeld })
      }}
      className='fixed bottom-6 right-6 w-16 h-16 p-3 rounded-full border-2 border-white bg-black shadow-lg'
    ></button>
  </>
)}
```

The touch controls reuse the same `keyDownHandler` and `keyUpHandler` functions, ensuring consistent behavior across input methods.

## Key Features

- **No Props Required**: The Player component is self-contained with no required props
- **Global State Integration**: Dispatches position updates to Redux for consumption by collision detection
- **Cross-Platform**: Automatically adapts to keyboard or touch input
- **Precise Collision**: Three-point hitbox system enables accurate collision detection with obstacles
- **Smooth Rotation**: 2-degree increments with 10ms updates provide fluid rotation animation
